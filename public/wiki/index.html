What is this project?

LMS Islam Fakultas is a web-based Learning Management System built for internal use by an Islamic faculty. It provides a complete digital learning environment where administrators manage users and courses, instructors build and deliver course content, and students learn, take quizzes, submit assignments, and earn certificates.

ℹ️
Scope

This is an internal faculty platform — not a public marketplace. It is designed for a single institution with a fixed set of roles: Admin, Instructor (Dosen), and Student (Mahasiswa).

Problem Statement

Faculty courses were managed through scattered tools — WhatsApp groups, Google Drive, and email. There was no unified platform for:

  • Structured course delivery with modules and materials
  • Online quizzes with automatic grading
  • Assignment submission and instructor feedback
  • Student progress tracking and certificates
  • Enrollment management with approval workflows

🏗️ System Architecture

Monolith modular — Laravel backend + Inertia.js bridge + React frontend in a single codebase.

High-Level Architecture

The application follows a monolithic modular architecture. Laravel handles routing, business logic, and data access. Inertia.js eliminates the need for a separate REST API by passing server-rendered props directly to React components. The entire app is deployed as a single unit.

Frontend Architecture

The frontend is a React SPA driven by Inertia.js. There is no client-side router — navigation is handled by Inertia's <Link> component which makes XHR requests and swaps page components without a full reload.

Queue & Scheduler

Notifications are dispatched to the database queue and processed by a queue worker. The scheduler runs daily at 08:00 to send assignment deadline reminders (H-1).

Coding Standards

PHP (Laravel)

  • PSR-12 code style, enforced by Laravel Pint
  • Use PHP 8.3 attributes for model fillable/hidden: #[Fillable([...])]
  • Controllers should be thin — delegate business logic to Services
  • Use Form Request classes for validation (app/Http/Requests/)
  • Use Policies for authorization, not inline abort(403)
  • Always eager-load relationships to avoid N+1 queries
  • Use query scopes for reusable filters (scopeActive, scopePublished)

JavaScript / React

  • Functional components only — no class components
  • Use Inertia's useForm hook for all form state
  • Use usePage() to access shared Inertia props (auth user, flash messages)
  • Component files: PascalCase (VideoPlayer.jsx)
  • Utility files: camelCase (utils.js)
  • Keep page components lean — extract reusable UI to components/

🔌 API Reference

All routes are server-rendered via Inertia.js. There is no separate REST API — controllers return Inertia responses or redirects.

ℹ️
Inertia.js Architecture

All routes return either Inertia::render() (page component + props) or redirect(). AJAX requests from Inertia include an X-Inertia: true header and receive JSON responses. Full-page requests receive the full HTML shell.

Authentication

GET /login

Render the login page. Redirects to dashboard if already authenticated.

Auth: Guest only (redirects if authenticated)
Returns: Auth/Login page component
POST /login

Authenticate user and create session. Rate limited to 10 attempts per minute.

Auth: Guest only | Rate limit: 10/min

Leaderboard Score

php// Best quiz score per quiz, then average
$quizScore = QuizAttempt::where('user_id', $userId)
    ->where('status', 'graded')
    ->whereHas('quiz', fn($q) => $q->where('is_published', true)
        ->whereHasMorph('quizzable', [Module::class], fn($mq) =>
            $mq->where('course_id', $courseId)
        )

    ->map(fn($attempts) => $attempts->max('score'))

jsx// ThemeContext.jsx

    const [theme, setTheme] = useState(
        () => localStorage.getItem('theme') || 'light'
    );
    const toggleTheme = () => {
        const next = theme === 'light' ? 'dark' : 'light';
        setTheme(next);
        localStorage.setItem('theme', next);
        document.documentElement.setAttribute('data-theme', next);
    };
    return <ThemeContext.Provider value={{ theme, toggleTheme }}>{children}</ThemeContext.Provider>;
};

⚡ Performance Optimization

Strategies for keeping the app fast at scale.

Backend Optimizations

Eager Loading (N+1 Prevention)

Always eager-load relationships when iterating over collections.

php// ❌ N+1 — fires a query per enrollment
$enrollments = Enrollment::where('course_id', $id)->get();
foreach ($enrollments as $e) { echo $e->user->name; }

// ✅ Eager load — 2 queries total
$enrollments = Enrollment::with('user')->where('course_id', $id)->get();

Query Optimization

  • Use select() to fetch only needed columns on large tables
  • Use withCount() instead of loading full relationships just for counts
  • Use database indexes on all foreign keys and frequently filtered columns
  • Use chunk() for bulk operations (e.g., sending notifications to all students)

  • Certificate template is a PNG stored in public/template-sertifikat.png
  • YouTube thumbnails are served directly from YouTube CDN — no local storage needed

Video Loading

The VideoPlayer component uses lazy loading — the YouTube iframe is only rendered when the user scrolls it into view, preventing unnecessary iframe loads on pages with multiple videos.

All notifications are queued to avoid blocking HTTP responses. The queue worker processes jobs asynchronously.

Development FAQ

How do I add a new role?

1. Add the role to the role enum in the users migration (requires a new migration). 2. Add role check methods to User model. 3. Create a new route group with role:newrole middleware. 4. Create corresponding controllers and pages.

How do I add a new content type?

1. Add the type to the type enum in the contents migration. 2. Update InstructorContentController to handle the new type. 3. Update config/lms.php with allowed MIME types if it's a file upload. 4. Add rendering logic in the student course view.

How do I add a new course setting?

1. Add the key to Course::getDefaultSettings(). 2. Update the settings form in Instructor/Courses/Settings.jsx. 3. Use $course->getSetting('key', $default) wherever the setting is needed.

How do I debug queue jobs?

bashphp artisan queue:failed          # List failed jobs
php artisan queue:retry all       # Retry all failed jobs
php artisan queue:retry {id}      # Retry specific job
php artisan queue:flush           # Delete all failed jobs
php artisan queue:work --once     # Process one job and exit (debug)

How do I reset a student's quiz attempt?

bashphp artisan tinker
>>> App\Models\QuizAttempt::where('user_id', $userId)
...     ->where('quiz_id', $quizId)
...     ->delete();

Microservice Readiness

The current monolith is well-structured for a future microservice extraction if needed. Natural service boundaries already exist:

  • Certificate Service — already isolated in CertificateService + CertificatePdfService
  • Notification Service — already queue-based, easy to extract
  • Video Service — stateless URL parser, trivial to extract
  • Leaderboard Service — compute-heavy, good candidate for caching layer

Before extracting microservices, ensure you have: centralized auth (JWT/OAuth), API gateway, distributed tracing, and a team large enough to maintain multiple services.
